# 5부 스레딩

## 26장 스레드의 기본

### 윈도우는 왜 스레드를 지원하는가?

 프로세스 - 응용 프로그램의 단일 인스턴스가 사용하는 리소스의 집합체이며, 개별 프로세스는 자신만의 가상 주소 공간을 가지고 있어서 다른 프로세스가 자신의 코드나 데이터에 접근할 수 없다.
시스템이 보안적으로 좀 더 안전할 수 있음.

 스레드는 CPU의 가상화하기 위한 운영체제의 개념이다. 개별 프로세스에게 각자의 스레드를 나누어 주는데, 이를 통해 특정 응용 프로그램이 무한 루프에 빠지게 되더라도 그 프로그래만 영향을 받고 다른 프로세스들은 계속 수행될 수 있도록 해줄 수 있다.

### 스레드의 비용

스레드는 다음에 나열하는 것들을 모두 하나씩 가지고 있다.

- 스레드 커널 객체
    - 고유의 스레드 데이터 구조체를 가지고 있다. 스레드 컨텍스트 정보가 있다. (CPU 레지스터 값을 저장) x86 : 700byte, x64 : 1240byte
- 스레드 환경 블록 (Thread environment block, TEB)
    - TEB는 유저 모드에 할당되고 초기화되는 메모리 블록. 1페이지 (4KB) 의 메모리를 사용하며 Thread Local Storage (TLS) 데이터를 가지고 있다.
- 유저 모드 스택
    - 지역 변수와 함수의 매개변수를 저장하는 용도. 현재 실행중인 함수가 반환될 때 그 다음으로 수행해야 하는 위치를 저장하고 있음. 대략 1MB 정도를 예약하고 필요에 따라 물리적 저장소를 커밋하기도 한다.
- 커널 모드 스택
    - 응용 프로그램이 커널 모드 함수로 매개변수를 전달해야 할 때 사용. 유저 모드 커널에서 커널 모드로 매개변수를 전달할때 유저 모드 스택의 내용을 커널 모드 스택으로 복사함. 커널 안에서 함수 호출할 때 함수 매개변수를 전달하기 위한 용도로 사용. x86 12KB, x64 24KB
- DLL의 스레드 attach / detach 통지
    - 프로세스 내에서 새로운 스레드가 생성되면 해당 프로세스의 메모리 공간에 로드된 모든 비관리 DLL 들의 DllMain 함수를 DLL_THREAD_ATTACH 플래그를 매개변수로 호출한다.

컨텍스트 전환(context switch) 비용

- 윈도우 운영체제는 하나의 CPU에 하나의 스레드를 할당한다. 스레드는 주어진 타임 슬라이스(Time slice, Quanntum 이라고도 함) 동안 수행된다.  타임 슬라이스 만큼 수행 완료 후 다른 스레드로 컨텍스트를 전환한다.
    1. CPU 레지스터의 값을 현재 수행 중인 스레드의 컨텍스트 구조체에 저장.
    2. 다음 수행할 스레드를 선택함. 선택한 스레드가 다른 프로세스가 있다면, 스레드가 수행할 코드와 데이터 접근을 위해 가상 메모리 주소를 우선 전환.
    3. 선택된 스레드의 컨텍스트 구조체 내의 값을 CPU 레지스터로 로드.
- 컨텍스트 전환 완료 후 다음 타임-슬라이스 만큼 스레드가 수행하고 다시 컨텍스트 전환을 반복해서 수행함. 윈도우 운영체제는 매 30밀리초 (0.03초) 마다 컨텍스트 전환을 수행한다.
