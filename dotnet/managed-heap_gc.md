## 관리 힙(Managed Heap)


### 관리 힙이란?

CLR 에서 관리하는 프로세스 내 할당 받아 사용할 수 있는 메모리 영역입니다.

### 관리 힙의 리소스 할당

1. 프로세스가 초기화되면 관리 힙 공간을 생성 후 할당 (32bit 최대 1.5GB, 64bit 최대 8TB) 됩니다.
2. new 연산자 호출시 힙 공간에 사용이 필요한 크기를 계산 (필드와, 상속 필드 정보 포함) 합니다.
3. 타입에 필요한 바이트 + type object 정보 + sync block index 정보가 최종 필요한 크기로 설정됩니다.

## 가비지 수집기(Garbage Collector)

### GC란?

관리 힙에 등록되어 있는 객체 중 참조 관계가 없다고 판단되는 객체 데이터를 삭제하는 기능입니다. 가장 최근에 생성된 객체부터 0세대, 1세대, 2세대로 구성되어 관리됩니다.

### GC 실행

1. 0세대의 허용된 공간이 부족한 경우
2. 코드에서 명시적으로 Collect를 호출한 경우
3. OS에서 메모리의 부족을 확인하였을 경우
4. 프로세스가 종료(shutdown)되었을 경우 (컴팩트 작업 제외)
5. 앱도메인이 내려(unload)가는 경우

### 참조 추적(reference tracking)

참조 타입의 변수를 대상으로 하는 알고리즘 입니다.  힙에 있는 객체와 연결(참조)되어 있는 객체를 판별하기 위한 기능입니다. GC 실행 후 marking phase 에서 사용됩니다.

### 마크과 컴팩트(marking phase, compacting phase)

GC가 실행되면 CLR은 프로세스 내 스레드를 모두 멈춘 후 마크와 컴팩트를 진행합니다.

** 마크는 참조 추적 알고리즘은 사용하여 을 통해 가비지 평가가 실행됩니다.

1. 힙에 등록되어 있는 객체의 sync block index 의 특정 비트를 모두 0으로 설정합니다. (삭제로 시작)
2. 힙에 등록되어 있는 객체를 순회하며 참조하고 있는 객체를 확인합니다.
3. 참조되어 있는 객체가 발견되면 참조 객체의 sync block index 의 특정 비트를 1로 변경합니다. (사용중)
4. 이 후 미확인된 모든 객체를 확인합니다. (확인 객체는 건너뜀)

** 컴팩트는 마크가 완료된 객체들을 힙의 시작지점쪽으로 이동하여 연속된 메모리 공간에 위치하도록 하는 기능입니다.

1. 힙의 메모리 단편화를 해결합니다.
2. 지역성이 살아나는 효과를 가지고 있습니다.
3. 워킹셋의 크기가 감소합니다.
4. 대형 객체는 속도 문제로 인해 컴팩트를 진행하지 않습니다.

컴팩트로 인해 객체의 메모리 주소가 변경됩니다. CLR은 해당 객체를 이동한 거리만큼을 빼주는 작업을 진행합니다.

### 세대를 고려한 GC (Generational garbage collector)

GC는 관리 힙의 메모리를 가장 최근에 생성된 객체를 기준으로 0세대, 1세대, 2세대로 그룹지어 관리합니다.

1. 0 세대는 가장 최근에 생성된 객체가 추가되며, 객체의 생성으로 인해 가용 공간이 부족한 경우 가비지 평가를 시작합니다.
2. 가비지라고 판단되는 객체에 한하여 수집을 진행하고, 빈 공간이 생기면 컴팩트를 진행합니다. 남아 있는 객체를 1세대로 승격하게 되며 0세대는 비어있게 됩니다.
3. 객체의 생성으로 0세대의 가용 공간이 부족한 경우 GC는 가비지 평가를 진행할 세대를 결정합니다. - 1세대의 가용공간이 남았다고 판단되면, 0세대의 가비지 평가를 진행하며, 2. 의 행동을 반복합니다.
4. 객체의 생성으로 0세대의 가용공간이 부족한 경우 가비지 평가를 시작하며, 1세대의 가용공간이 적다고 판단되면, 1세대의 가비지 평가를 진행합니다. 남아 있는 객체들은 2세대로 승격하게 되며, 이후 0세대의 가비지 평가를 진행하기 위해 2. 의 행동을 반복합니다.

### 대형 객체 관리 (LOH - Large Object Heap)

CLR은 85kb 이상의 객체를 대형 객체로 관리합니다. (.net 4.5 기준)

1. 힙에서 관리하지 않으며, 프로세스 주소 내 다른 공간에서 별도 관리합니다.
2. 생성 즉시 2세대로 처리됩니다.
3. GC의 컴팩트에서 제외됩니다.

## 그외

### 참조의 지역성 (locality of Reference)

유사 시점에 생성된 객체들은 연관성을 가지고 빈번한 참조를 가지를 경향을 가지고 있습니다.

연관된 참조로 인해 CPU 캐시에 같이 올라가서 계산되는 효율을 발휘할 수 있기 있습니다. CPU 캐시로부터 값을 가지고 올 수 있다면, 캐시 미스(Cache Miss) 로 인해 RAM에 접근하지 않고 빠르게 처리 할 수 있습니다.

### 각 세대별 허용 크기

CLR은 관리 힙을 초기화할 때, 세 개의 세대에 개별적으로 허용된 메모리 크기를 부여합니다.

가비지 수집기는 자가 조정(self-tuning) 수집기이므로 응용 프로그램에서 가비지 수집이 일어날때마다 자체적으로 학습하여 세대별 메모리 크기를 조정합니다.
